## 第零阶段：零成本，立即可做

### 改进 10：ToMe Token 合并

**做什么**：在 EVA ViT 的 Transformer Block 中插入 Token Merging（ToMe），用双向软匹配合并相似 token，无需修改任何权重。

**为什么先做**：
- 免训练，当天可完成
- 改善历史帧信息保留，消除当前固定阈值（0.985）的场景不适应问题
- 吞吐量提升约 2×，评测更快

**验收标准**：插入 ToMe 后跑一次 val_unseen 评测，确认 SR 损失 < 0.5%。若损失可接受则保留，否则调小合并比例。

**参考**：[Token Merging: Your ViT But Faster](https://arxiv.org/abs/2210.09461)，ICLR 2023。

## 第一阶段：核心模型升级

### 改进 1 + 改进 6：LLaMA-3.1-8B × 课程学习（合并一次训练）

**为什么合并**：改进 6 只改变数据采样顺序，不影响模型结构，可以在改进 1 的训练中同时启用，**一次训练得到两个改进的效果**，无需额外训练。

#### 改进 1：LLaMA-3.1-8B 基座替换

**做什么**：将语言模型从 Vicuna-7B（LLaMA-2 基座）替换为 LLaMA-3.1-8B。

**预期收益**：
- 指令理解和空间推理能力全面提升（MT-Bench 6.57 → 8.22）
- 上下文从 4K 扩展至 128K，支持更长导航历史
- 架构与现有代码高度兼容，改动极小

**操作步骤**：
```bash
# 1. 下载模型权重
# meta-llama/Meta-Llama-3.1-8B-Instruct

# 2. 修改训练脚本中的基座路径
# scripts/uninavid_stage_1.sh 中 PREV_MODEL 替换为新路径

# 3. 升级 transformers
pip install "transformers>=4.40"

# 4. 重新运行 Stage 1 + Stage 2 训练
```

#### 改进 6：课程学习数据采样

**做什么**：修改 `LazySupervisedDataset` 的采样策略，将训练样本按路径难度（步数、转弯次数）从易到难排列，取代当前随机采样。

**预期收益**：训练收敛速度加快约 20-30%，最终精度与随机采样持平或略优。

**操作步骤**：
```python

# 难度度量：路径步数 / 转弯次数 / 指令词数
# 训练前期用简单样本，后期逐步引入复杂样本
```

**训练成本**：Stage 1 + Stage 2 完整训练一次，与原始训练周期相同。

## 第二阶段：DPO 偏好对齐

### 改进 2：SFT + DPO

**做什么**：在第一阶段 SFT checkpoint 基础上，继续用 DPO 做偏好对齐训练。

**为什么效果确定**：
- SFT 只教模型"正确答案是什么"
- DPO 教模型"成功轨迹比失败轨迹好在哪里"，更接近导航的真实目标
- 预期 SR 提升 **+3~8%**

**数据准备**（在第一阶段训练期间同步收集）：

```
数据格式（三元组）：
  prompt:   起点位置 + 导航指令
  chosen:   成功到达目标的动作序列
  rejected: 走错路或超时失败的动作序列

数据来源：
  从 VLN-CE 仿真器评测日志中直接提取
  tmp/xxx/log/*.json 中 success=1 的为 chosen，success=0 的为 rejected
```

**操作步骤**：
```bash
# 使用 HuggingFace TRL 库
pip install trl

# DPOTrainer 接受标准三元组格式，无需修改模型结构
# 在第一阶段 checkpoint 上 fine-tune
```

**训练成本**：小时级，远低于重新全量训练。

## 第三阶段：视觉理解升级

### 改进 8：SigLIP-SO400M 编码器替换

**做什么**：将视觉编码器从 EVA ViT-g（1408 维）替换为 SigLIP-SO400M（1152 维）。

**预期收益**：
- 图文对齐质量提升（Sigmoid Loss 优于 InfoNCE）
- 参数量从 ~1B 降至 ~400M，推理更快
- 场景识别和物体匹配更准确

**操作步骤**：
```
1. 新增 siglip_encoder.py，封装 SigLIP 编码器
2. 修改 multimodal_encoder/builder.py 路由
3. 调整 mm_projector 输入维度：1408 → 1152
   config.mm_hidden_size = 1152
4. 重新训练 Stage 1（视觉编码器 frozen，约 1-2 小时）
   无需重跑 Stage 2（直接基于 Stage 1 新 checkpoint）
```

**⚠️ 注意**：必须在第一阶段（确定 LLM 基座）之后做，否则 Stage 1 需重跑两次。

### 可选叠加：改进 13（DINOv2-ViT-L）

若改进 8 完成后 SR 仍有提升空间，可在 SigLIP 基础上叠加 DINOv2 作为第二编码器，专门处理空间定位（深度感知、物体位置、场景结构）。对 ObjectNav 任务尤为有效。

## 长期规划

### 改进 18：PPO / GRPO 强化学习

**做什么**：以导航成功率（SR）和路径效率（SPL）为奖励，在 Habitat-Sim 仿真环境中进行在线强化学习。

**预期收益**：突破 SFT + DPO 的天花板，在 VLN-CE R2R benchmark 上有望进入 SOTA。

**前置条件**：
- 搭建 Habitat-Sim 在线仿真环境
- 设计奖励函数（稀疏 SR + 稠密距离奖励）
- 需要 8-16 卡 GPU 资源

**参考**：EmbodiedGPT、NavRL 等工作。

## 注意事项

1. **阶段顺序不能颠倒**：改进 8（SigLIP）需在改进 1（确定 LLM 基座）之后做，否则 Stage 1 要额外重跑一次。

2. **ToMe 要先验证精度**：插入 ToMe 后立即跑一次 val_unseen，确认 SR 损失 < 0.5%，再继续后续阶段。

3. **DPO 数据要提前收集**：第一阶段训练期间，同步跑评测并保存失败轨迹日志，第二阶段可以立即开始，不需要额外等待。

4. **每个阶段完成后先评测再进入下一阶段**：每个 checkpoint 都保存在 `model_zoo/` 下，用 `compare_results.py` 对比增量效果，确认有提升再继续投入。
